What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. Continuous Integration (CI) is the practice of automating the integration of code changes from multiple contributors into a single software project. Continuous Deployment (CD) is the process of automatically deploying those changes to a production environment, like GitHub Pages. This approach helps developers find bugs faster and deliver updates to users more frequently and reliably without manual intervention.

My Pipeline

Once I push my code to the main branch on GitHub, a GitHub Actions workflow is triggered. It first checks out the code using "actions/checkout", verifies that all essential files (HTML, CSS, JS) exist, and then automatically deploys the updated site to GitHub Pages using "actions/deploy-pages".
